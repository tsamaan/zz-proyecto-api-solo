# Informe de estructura de carpetas inicial

## Estructura creada

- `/docs`: Documentación y reportes de avance.
- `/backend`: Carpeta para el backend simulado y futuro backend real.
  - `/backend/json-server`: Para el mock de API REST con json-server.
- `/src/components`: Componentes reutilizables de React.
- `/src/services`: Servicios para conexión con APIs y lógica de negocio.
- `/src/context`: Contextos globales de React (usuario, carrito, etc.).
- `/src/pages`: Vistas principales (Home, Detalle, Carrito, etc.).
- `/src/hooks`: Hooks personalizados.
- `/src/utils`: Utilidades y helpers.
- `/src/assets/img`: Imágenes y recursos gráficos.
- `/src/assets/styles`: Estilos globales y parciales.

## Decisiones
- Se priorizó la escalabilidad y el orden, separando frontend y backend.
- Se preparó la estructura para trabajar con React, React Router, hooks y contextos.
- Se creó la carpeta `docs` para registrar cada avance y decisión.
